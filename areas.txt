REPOSITORY: (committed)
(directorio git)




Modified:
    Si la versión del archivo contiene cambios que no son parte del repositorio
    Y NO se ha añadido al staging AREA

Staged:
    Si la versión del archivo contiene cambios que no son parte del repositorio
    Y SI se ha añadido al staging AREA

Committed:
    YA se añadió al directorio git




Commit: 
    Es un registro o foto del estado de un proyecto en un momento especifico
    Registra los cambios que se realizaron en un archivo
    Cada commit añada un SHA (identificador único)



Branch:
    Linea independiente de desarrollo en un repositorio, sin afectar la que está en vivo
    Crear:
    git branch <nombre-de-la-rama>

    Crear y moverse:
    git checkout -b <nombre-de-la-rama>

    Moverse es con SWITCH:
    git switch <nombre-de-la-rama>
    

    ## checkout lo que hace es descargar 

    si edito un archivo, lo guardo, pero no lo stage, puedo devolverme con
    git checkout <nombredelarchivo>


    Cambiar nombre estando en ella:
    git branch -m <nuevo-nombre>

    Sin estar en ella:
    git branch -m <nombre-viejo> <nombre-nuevo>

    Eliminar branch:
    git branch -d <nombre-de-la-rama>

    DESDE DONDE ESTÉ ES QUE SE CREA Y COPIA LA RAMA



Merge:
    Combinar varias lineas de desarrollo en una sola rama (combinar branches)

DEBO ESTAR EN LA RAMA QUE RECIBIRÁ LA FUSIÓN

git merge <rama-que-voy-a-traer>



Stash:
git stash
git stash list
git stash pop
Es un commit local que no afecta al arbol.




Comandos:

git add <file>
git rm --cached <file>..." to unstage
git commit -m "texto"
git commit --amend 


Configuraciones:
Para cambiar el editor por defecto:
git config --global core.editor "code --wait"

Para crear un alias de un comando mas largo
git config --global alias.<nombredelalias> "<comando sin la palabra git>"


Comparar lo que se ha cambiado, guardado, pero no staged:
git diff 

git log -p
git log --oneline

Para devolverse a un estado anterior violentamente
git reset --hard <HASH>

Deshago el commit, pero los cambios que habia hecho se mantienen y quedan staged
git reset --soft HEAD~1

Historial completo de interacciones
git reflog


Tag para etiquetar milestones:
git tag "<texto>"


Proceso:

git status (the status)
git add . (stage a file)
git commit (commit)
git log (log)




GITHUB:

La prmera vez que se van a traer cambios de un repositorio remoto toca seleccionar qué hacer
preferiblemente merge

Descarga el historial de cambios, pero sin descargar los cambios:
git fetch

Descarga el historial de cambios y  los cambios:
git pull


git push
